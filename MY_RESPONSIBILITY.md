# ВАЖНЫЕ ПРАВИЛА (КРИТИЧЕСКИ ВАЖНО)

## Правило #1: НИ ОДНОГО изменения без явного разрешения пользователя

**БЕЗ ЯВНОГО РАЗРЕШЕНИЯ ПОЛЬЗОВАТЕЛЯ НЕ ВНОСИТЬ НИ ОДНО ИЗМЕНЕНИЕ:**
- В код бота (.py файлы)
- В .env конфиги
- В state файлы
- В cron, systemd, screen
- В любые настройки VPS

**Разрешённые действия без подтверждения:**
- Чтение логов, state, конфигов (read-only)
- Перезапуск бота в случае падения (screen -dmS ...)
- Аварийный рестарт после ребута VPS
- Backfill сделок, генерация дашборда

**Запрещённые действия без подтверждения:**
- Любые правки кода/конфигов
- Изменение параметров стратегии (SL, TP, ADX, cooldown и т.д.)
- Включение/выключение фич
- Изменение interval/cooldown значений
- Удаление файлов

**Нарушение = потеря доступов.**

## Правило #2: Разделение ответственности

- **GOLD бот** (cTrader $100K demo) — МОЯ ответственность. Изменяю только с разрешения.
- **BTC бот** (cTrader $5K demo) — НЕ ТРОГАТЬ ВООБЩЕ. Только анализировать статус.
- **HL paper боты** (GOLD, BTC, HYPE, XYZ100, WTI) — только анализировать.

## Правило #3: Всегда перезапускать screen после ЛЮБЫХ изменений кода/.env

После правки кода или .env — обязательно:
1. Перезапустить screen сессию бота
2. Проверить в логе что новые настройки применились
3. Запушить изменения в GitHub

## Правило #4: Никогда не говорить "всё в порядке" без проверки реальных сделок

Перед любым утверждением о статусе — проверить:
- Реальные сделки в trade log
- State файл
- Баланс в cTrader
- Открытые позиции

Одного снимка ADX/ATR недостаточно — нужны реальные PnL.
